About The Game

To-Gather is a Co-Op Survival Story Game I've been working on with 3 other people who follow the same study. We are planning on putting this game on steam, so far it's all going according to plan and you'll soon be able to purchase the game!

My Work

Things I've Worked On

  • Player Info Script - General Info Storage for each player
  • Player Movement Script - Responsible for all the player movement
  • Inventory, Held Item & Gear Script - All the 'Storages' the player has
  • Building System - System used for building all the buildables in the game
  • Crafting System - System used for all the crafting in the game
  • Interact System - System that manages all the interactable things in the scene, excluding pickups
  • UI Manager - Manager used for UI so it doesn't overlay each other & to manage binds
  • Tool Tips - Item Tooltips displayed when hovering over the items
  • Pickup Manager - Manages which item you're gonna pick up based on distance & displays a particle making the item more visable
  • Water Suffucation - Makes you lose health over time when underwater for too long
  • Water Muffle & Blur - Blurs your vision & Muffles the sound of the game making it feel like you truly are under waters
  • WIP Enemy Heatmap - Wandering System for the Enemies based on where you've been active the most throughout the day

    • Inventory & Item System

    The Inventory & Item System are very adjustable. The Base of the System is called 'Container', Container is a general storage for 'Resources', making it easily expandable for things such as an Inventory or Chest. In the Container Script there are a couple options. Slots, which is a collection containing all the Slots in a List. The reason this is a list is for expansion, with in mind the idea that the inventory can always be expanded with an extra slot for lets say a Backpack. Other than that there's also a checkbox called Online. This is used to Sync the items over network.

    Now the Item System. The Item System works as following; There's 1 script called Resource. Resource is a ScriptableObject containing the Name of the item, a Description of the item, the Maximum Amount of how many one stack can contain and the Type of item, which is used for Specific Slots such as a Head Gear. Other than those variables the script also contains three methods; GetSprite(), GetPickupPath(), and GetHeldPath(). These methods all work in a simular way, by getting the wanted type by Path automatically, so you don't have to manually assign them each time you make an item, making it easier for everyone involved to create the wanted Resource.

    Tooltips

    Now the Tooltips, the Tooltips are quite simple and work by shooting a Raycast at the position of the mouse, checking all the hit UI Elements for a Slot Script. If the Slot Script has been found it'll check if the Slot has an Item, if so It'll grab the Display Name and set the Text Element's text to the Display Name. The 'Held Item' slot won't display the Tooltip, because the tooltip ignores Raycast on the Held Item slot
